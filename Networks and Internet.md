## 第一篇：網路（Network）——基礎架構與連線技術

---

## 一、網路基本概念

### 1. 什麼是網路？

**網路（Network）** 是指兩台或兩台以上的裝置，透過某種傳輸媒介（有線或無線）相互連接，以便交換資料、共享資源的系統。網路中的每一個連接裝置，稱為**節點（Node）**，可以是電腦、印表機、感測器、手機，甚至是生產機台。

最簡單的網路只有兩台電腦直接相連；最複雜的網路則是將全球數十億台裝置互相連結的網際網路。不論規模大小，所有網路的核心目的都相同：**讓裝置之間能夠溝通與共享**。

### 2. 網路的功能

網路的存在帶來了幾項根本性的能力：

- **資料傳輸（Data Transfer）**：將資訊從一個裝置傳遞到另一個裝置，例如感測器將量測值傳送到控制系統
- **資源共享（Resource Sharing）**：多台裝置共用同一台印表機、同一個資料庫或同一套軟體授權，降低成本
- **集中管理（Centralized Management）**：透過網路，管理人員可以從單一位置監控、設定並更新多台遠端裝置
- **協同作業（Collaboration）**：不同地點的人員可以即時存取同一份資料，共同完成工作

### 3. 網路 vs 網際網路（概念區分）

初學者常將「網路」與「網際網路」混為一談，但兩者有本質上的差異：

| 面向 | 網路（Network） | 網際網路（Internet） |
| --- | --- | --- |
| 範圍 | 單一或局部連線（如一棟建築、一個工廠） | 全球數十億台裝置的互聯系統 |
| 所有權 | 通常由單一組織擁有與管理 | 無單一所有者，由各國各組織共同維運 |
| 比喻 | 一棟大樓的內部電話系統 | 全球公共電話網路 |

簡單來說：**工廠內的設備連線是「網路」；工廠的系統連到雲端平台，則需要透過「網際網路」**。本篇先深入了解「網路」的基礎，下一篇再討論「網際網路」。

### 4. 工業應用：工廠內部設備連線

現代工廠的運作高度依賴內部網路。一個典型的製造工廠中，網路連接了：生產機台（CNC 加工中心、機器人手臂）、各種感測器（溫度、壓力、振動）、PLC 控制器、HMI 操作介面、MES 伺服器，以及工程師的工作電腦。

這些設備透過工廠內部網路相互溝通，使得感測器資料能即時傳入控制系統、控制指令能立即下達到機台、生產紀錄能自動上傳到 MES——這一切都仰賴穩定、低延遲的內部網路架構。

## 二、網路拓樸（Topology）

### 1. 什麼是網路拓樸？

**網路拓樸（Network Topology）** 是描述網路中各節點如何相互連接的幾何佈局方式。拓樸的選擇，決定了網路的可靠性、擴充性、成本與維護難度。就像工廠的廠房配置影響物料流動效率，網路拓樸影響資料流動的效率與韌性。

### 2. 星狀拓樸（Star Topology）

所有節點都連接到一個中央裝置（通常是**交換器 Switch**），節點與節點之間不直接相連，所有通訊都經由中央裝置轉發。

```
    設備A
      │
設備B─┼─ Switch ─ 設備C
      │
    設備D
```

- **優點**：任一節點故障不影響其他節點；新增節點容易；故障診斷簡單
- **缺點**：中央交換器是單點故障（SPOF）——若交換器故障，整個網路癱瘓；纜線用量較大

星狀拓樸是**現代辦公室與工廠內部網路最常見的選擇**，因為其管理簡便、擴充容易。

### 3. 匯流排拓樸（Bus Topology）

所有節點共用一條主幹線（匯流排），訊號沿著主幹線傳播，每個節點都能接收到，但只有目標節點會處理該訊號。

```
設備A ─ 設備B ─ 設備C ─ 設備D
│─────────────────────────│
              主幹線
```

- **優點**：佈線簡單、成本低
- **缺點**：主幹線斷線即全線癱瘓；節點越多，碰撞與延遲越嚴重

匯流排拓樸在早期以太網路廣泛使用，現今已少見於一般辦公網路，但在某些**工業現場匯流排（Fieldbus）**中仍有應用。

### 4. 環狀拓樸（Ring Topology）

每個節點只與前後相鄰的節點連接，形成一個封閉的環形。資料沿著環形依序傳遞，直到到達目的節點。

```
設備A ─→ 設備B ─→ 設備C
  ↑                    ↓
設備D ←─────────────────
```

- **優點**：每個節點機會平等，不容易發生碰撞；可預測的資料傳輸時間（對即時控制有優勢）
- **缺點**：任一節點或連線故障可能中斷整個環；新增節點需要中斷網路

環狀拓樸在某些工業通訊協定（如 PROFIBUS 的部分應用）中使用，因其可預測的延遲特性對即時控制友善。

### 5. 網狀拓樸（Mesh Topology）

每個節點都與多個（甚至全部）其他節點直接相連，資料可以選擇多條路徑到達目的地。

- **完全網狀（Full Mesh）**：每個節點與所有其他節點直接連接，可靠性最高
- **部分網狀（Partial Mesh）**：只有關鍵節點間有多條連接路徑

- **優點**：極高的容錯性——即使多條連線故障，資料仍可繞道傳輸；適合高可靠性需求
- **缺點**：佈線複雜、成本極高，管理困難

網狀拓樸用於**對可靠性要求極高的關鍵基礎設施**（如電力公司的控制網路、骨幹網路）；工業無線感測網路（WSN）也常採用網狀結構，讓感測器節點互相中繼訊號，擴大覆蓋範圍。

### 6. 工業應用：產線設備配置

實際工廠的網路通常是**混合拓樸**：

- 辦公區與工程師站採星狀拓樸，連接到核心交換器
- 產線上的感測器採匯流排或環狀拓樸，與 PLC 連接
- 關鍵控制系統之間可能採部分網狀，確保高可靠性

選擇拓樸時，工業工程師需要權衡：設備數量、可靠性要求、佈線成本、維護難度，以及未來擴充的彈性。

## 三、網路硬體設備

### 1. 網路卡（NIC）

**網路介面卡（Network Interface Card，NIC）** 是讓裝置能夠連接網路的硬體元件，每台需要連網的裝置都必須配備 NIC。NIC 負責將裝置產生的資料，轉換為可在網路媒介上傳輸的電子訊號（有線）或無線電波（無線），並反向將接收到的訊號還原為資料。

每張 NIC 都有一個全球唯一的**MAC 位址（Media Access Control Address）**，由 48 個位元組成，通常以十六進位表示（如 `3C:22:FB:A1:0D:5E`）。MAC 位址就像裝置的「身分證號碼」，用於在區域網路中識別特定裝置。

### 2. 交換器（Switch）

**交換器（Switch）** 是區域網路中的核心連接裝置，提供多個連接埠，讓多台裝置能夠相互連通。交換器的智慧之處在於：它會學習並記錄每個連接埠所對應的 MAC 位址，當收到一個資料框時，**只將資料傳送到目的裝置所在的連接埠**，而不是廣播給所有人。這大幅降低了網路流量，提高效率。

**交換器 vs 集線器（Hub）的差異**：舊式的集線器會將收到的訊號廣播到所有連接埠，效率低且容易造成碰撞；交換器則精確傳送，是現代網路的標準選擇。

### 3. 路由器（Router）

**路由器（Router）** 負責在**不同網路之間**轉發資料，是網際網路運作的核心設備。若說交換器是管理「同一棟大樓內部的通訊」，路由器就是負責「不同城市、不同國家之間的郵件分揀與轉送」。

路由器依據資料封包中的**IP 位址**，決定最佳的轉發路徑。家用路由器同時兼具多重功能：連接家庭內部網路（LAN）與寬頻網路（WAN）、提供 Wi-Fi 存取、執行防火牆功能、以及透過 DHCP 自動分配 IP 位址給家中各裝置。

### 4. 存取點（Access Point，AP）

**無線存取點（Access Point）** 將有線網路延伸為無線網路，讓 Wi-Fi 裝置能夠加入有線網路。在大型場所（如工廠、醫院、購物中心），通常部署多個 AP，確保無線訊號的完整覆蓋。

### 5. 閘道（Gateway）

**閘道（Gateway）** 是連接兩個使用**不同通訊協定**的網路的橋梁。它不只是轉發資料，還要進行協定轉換，讓原本「說不同語言」的系統能夠溝通。

**工業應用**：工廠的 PLC 使用 Modbus 或 PROFIBUS 等工業通訊協定，而上層的 MES 系統使用標準的 TCP/IP 網路協定。工業閘道器負責在兩者之間進行協定轉換，讓 PLC 的資料能夠順利傳入 IT 系統，這正是工業4.0「OT/IT 整合」的關鍵硬體節點。

### 6. 工業應用：設備連網與資料收集

在智慧工廠中，上述各種硬體設備協同運作：NIC 讓每台機器具備連網能力、工業交換器在現場層傳遞 PLC 與感測器資料、工業閘道器轉換協定、路由器連接工廠網路與企業廣域網路、AP 讓移動中的操作員可用平板電腦存取系統。整個硬體架構，是智慧製造資料流的實體骨架。

## 四、區域網路（LAN）

### 1. 什麼是區域網路？

**區域網路（Local Area Network，LAN）** 是在有限地理範圍內（如一個房間、一棟建築、一個工廠廠區）連接多台裝置的網路。LAN 的特點是範圍有限但傳輸速度快、延遲低、管理集中，通常由單一組織擁有並維護。

### 2. 有線區域網路（Ethernet）

**乙太網路（Ethernet）** 是目前有線區域網路的主流標準，由 IEEE 802.3 規範制定。乙太網路使用雙絞線（如常見的 Cat5e、Cat6 網路線）或光纖作為傳輸媒介，透過 RJ-45 接頭連接到交換器。

乙太網路的傳輸速度從早期的 10 Mbps（每秒百萬位元），演進到目前工業環境常用的 100 Mbps（Fast Ethernet）、1 Gbps（Gigabit Ethernet），乃至資料中心使用的 10/25/100 Gbps。

工業環境中，使用的是特別強化的**工業乙太網路（Industrial Ethernet）**，在標準乙太網路的基礎上，加強了抗電磁干擾能力、擴大了操作溫度範圍、提高了連接器的防護等級（IP 等級），以應對工廠的惡劣環境。

### 3. 基本網路設定概念

一台裝置要加入區域網路，通常需要幾個基本設定：

- **IP 位址（IP Address）**：裝置在網路中的地址，讓其他裝置知道如何「找到它」（詳見第二篇）
- **子網路遮罩（Subnet Mask）**：定義哪些 IP 位址屬於同一個網路
- **預設閘道（Default Gateway）**：當資料需要傳到網路外部時，先送往哪裡（通常是路由器的 IP）
- **DNS 伺服器**：將網域名稱轉換為 IP 位址的服務（詳見第二篇）

這些設定可以手動配置，也可以透過 **DHCP（Dynamic Host Configuration Protocol）** 由網路自動分配，大幅簡化了管理工作。

### 4. 工業應用：工廠內部網路（內網）

工廠的內部網路（內網）通常與外部網際網路實體隔離或嚴格管控，稱為**工廠局域網路（Factory LAN）**。這種隔離設計有兩個主要目的：**安全性**（防止外部攻擊入侵工業控制系統）與**可靠性**（不受外部網路波動影響，確保生產控制的穩定）。

工廠內網通常按功能劃分為多個區段（Segment），例如：辦公室區段、工程師站區段、現場控制區段，各區段之間透過防火牆或 VLAN（虛擬區域網路）加以隔離，遵循資安的「縱深防禦」原則。

## 五、無線網路（Wireless Networks）

### 1. Wi-Fi 基本原理

**Wi-Fi** 是無線區域網路（Wireless LAN，WLAN）的通用名稱，基於 IEEE 802.11 系列標準。Wi-Fi 使用無線電波在裝置與無線存取點（AP）之間傳輸資料，讓裝置無需實體纜線即可連接網路。

Wi-Fi 標準持續演進，每代帶來更高的速度與更好的效能：

| Wi-Fi 世代 | 標準 | 最高速度 | 主要特性 |
| --- | --- | --- | --- |
| Wi-Fi 4 | 802.11n | 600 Mbps | 雙頻（2.4/5 GHz），MIMO 技術 |
| Wi-Fi 5 | 802.11ac | 3.5 Gbps | 5 GHz 專用，MU-MIMO |
| Wi-Fi 6 | 802.11ax | 9.6 Gbps | 高密度環境優化，低延遲，省電 |
| Wi-Fi 6E | 802.11ax | 9.6 Gbps | 新增 6 GHz 頻段，干擾更少 |

Wi-Fi 使用**2.4 GHz** 與 **5 GHz** 兩個主要頻段：2.4 GHz 穿透力強但速度較慢，適合大範圍覆蓋；5 GHz 速度快但穿透力弱，適合短距離高速傳輸。

### 2. 無線網路安全性

無線訊號在空中傳播，任何在訊號範圍內的裝置理論上都能接收，因此**安全性是無線網路的重要課題**。

**WPA3（Wi-Fi Protected Access 3）** 是目前最新的 Wi-Fi 安全標準，提供更強的加密（256 位元加密）與更好的防護（即使密碼被破解，歷史通訊仍受保護），是新建網路的推薦選擇。較舊的 WEP 加密已被證實不安全，不應再使用。

其他重要的無線安全措施包括：更改路由器預設密碼、隱藏 SSID（網路名稱）、啟用 MAC 位址過濾，以及定期更新路由器韌體。

### 3. 工業應用：無線感測網路（WSN）

**無線感測網路（Wireless Sensor Network，WSN）** 是由大量低功耗的無線感測節點所組成的網路，每個節點具備感測（如溫度、濕度、震動）、簡單運算與無線通訊的能力。

WSN 特別適合下列工業場景：感測節點分布廣泛而佈線困難的大型廠區；移動設備（如 AGV 自動導引車）或產品在生產過程中的追蹤；舊廠翻新時，無法輕易在現有設備上加裝有線網路的情況。

WSN 的主要挑戰是電池壽命與訊號可靠性，因此工業 WSN 通常採用低功耗的通訊協定（如 Zigbee、LoRaWAN）並設計節能的運作模式。

## 六、廣域網路（WAN）

### 1. 什麼是廣域網路？

**廣域網路（Wide Area Network，WAN）** 是跨越較大地理範圍（城市、國家乃至全球）的網路，通常由電信業者提供，企業租用其骨幹線路來連接分散各地的分支機構。網際網路本身可以視為全球最大的廣域網路。

WAN 的傳輸速度通常低於 LAN，延遲也較高（因為資料需要傳輸更遠的距離），但現代光纖骨幹網路已大幅改善了這些限制。

### 2. WAN 概念與應用

企業使用 WAN 的主要方式有幾種：

**專線（Leased Line）**：向電信業者租用一條點對點的專用線路，確保固定頻寬、低延遲與高可靠性，但費用昂貴。適合對連線品質要求極高的金融機構或大型製造企業。

**MPLS（多協定標籤交換）**：一種高效的 WAN 技術，讓企業的多個分支機構透過電信業者的骨幹網路互聯，具有服務品質（QoS）保障。

**VPN（虛擬私人網路）**：在公共的網際網路上建立一條加密的「虛擬專線」，讓遠端工作者或分支機構能安全地存取總部資源。成本遠低於實體專線，是中小企業常用的選擇。

### 3. 工業應用：跨廠區資料整合

跨國製造企業的多個工廠、研發中心、倉儲中心分布在不同城市甚至不同國家，需要透過 WAN 將各地的 ERP、MES、品質系統資料整合到集團總部，供管理層進行跨廠區的效能比較與資源調度。

例如，一家台灣的零件製造商，在台灣、越南、墨西哥各有工廠，透過 MPLS 或 SD-WAN 將三地的生產資料即時整合，台灣總部的管理層可以即時看到全球三地的產能與良率狀況，做出全球性的產能調配決策。

## 七、行動通訊技術（1G–5G）

### 1. 行動通訊的世代演進

行動通訊技術歷經數十年演進，每一代（Generation，G）都帶來傳輸速度、容量與功能的重大躍升：

**1G（1980年代）**：第一代類比行動通訊，只能傳輸語音，通話品質差、容量小，無加密保護。台灣於1989年開放 1G 服務。

**2G（1990年代）**：轉為數位訊號，語音品質提升，並開始支援簡訊（SMS）與低速資料傳輸（GPRS/EDGE，速度約 10–384 kbps）。這個時代催生了 LINE 的前身——簡訊文化。

**3G（2000年代）**：速度大幅提升至數 Mbps，讓智慧型手機能夠瀏覽網頁、收發電子郵件、使用行動應用程式，開啟了行動網際網路時代。

**4G LTE（2010年代）**：速度達到數十至一百多 Mbps，讓串流影音（YouTube、Netflix）、視訊通話成為日常，也使得行動支付與各種 App 服務蓬勃發展。

**5G（2020年代起）**：目前最新一代，帶來三大革命性特性：

| 特性 | 數字 | 意義 |
| --- | --- | --- |
| 超高速（eMBB） | 最高 20 Gbps | 4G 的 20 倍，超高清影像即時傳輸 |
| 超低延遲（URLLC） | < 1 毫秒 | 工業機器人即時控制、自動駕駛 |
| 大規模連接（mMTC） | 每平方公里 100 萬台裝置 | 大規模 IoT 感測器部署 |

### 2. 工業應用：智慧製造與即時監控

5G 的**超低延遲**特性，使其成為工業自動化的關鍵使能技術。傳統 Wi-Fi 的延遲約 10–50 毫秒，而工業控制（如機器人協作、精密切削的即時反饋）往往需要小於 5 毫秒的延遲——這正是 5G 所能提供的。

**私有 5G 網路（Private 5G）** 是近年智慧製造的重要趨勢：企業在工廠內部部署專屬的 5G 基地台，建立只服務本廠的私有 5G 網路。相比公共 5G 網路，私有 5G 提供更高的安全性（資料不離廠）、更可預測的服務品質，以及更大的客製化彈性。

## 八、個人網路與短距離通訊

### 1. 個人區域網路（PAN）

**個人區域網路（Personal Area Network，PAN）** 是以個人為中心、覆蓋範圍極小（通常在 10 公尺以內）的網路，用於連接個人隨身裝置。最典型的例子是手機透過藍牙連接耳機、手錶、鍵盤等周邊設備。

### 2. 藍牙（Bluetooth）

**藍牙（Bluetooth）** 是最廣泛使用的短距離無線通訊技術，使用 2.4 GHz 頻段，覆蓋範圍通常在 10 公尺以內（Class 1 藍牙可達 100 公尺）。

**藍牙低功耗（Bluetooth Low Energy，BLE）** 是藍牙的省電版本，大幅降低了功耗，特別適合需要長期運作的電池供電裝置，如工業用的資產標籤、穿戴式設備。BLE 感測器每次發送資料的耗電量極低，一顆鈕扣電池甚至可以支撐數年的運作。

### 3. NFC（近場通訊）

**NFC（Near Field Communication，近場通訊）** 是一種極短距離（4 公分以內）的無線通訊技術，通常用於「碰一下」就能完成的快速資料交換或感應。

日常生活中，NFC 的應用包括：悠遊卡與門禁卡（感應刷卡）、手機行動支付（Apple Pay、Google Pay）、電子票券。在工業場合，NFC 可用於工具或設備的快速身份識別：技術員用手機輕碰機台上的 NFC 標籤，立即讀取該機台的保養記錄、操作手冊或最近的警報歷史。

### 4. RFID 基本原理

**RFID（Radio Frequency Identification，無線射頻識別）** 是利用無線電波進行非接觸式識別的技術，由**標籤（Tag）** 和**讀取器（Reader）** 組成：

- **被動式 RFID 標籤**：本身無電池，靠讀取器發出的無線電波感應供電後回傳資料。成本低、體積小、壽命長，但讀取距離短（通常 < 1 公尺），需在讀取器附近才能被讀取。
- **主動式 RFID 標籤**：內建電池，主動定期發送訊號，讀取距離可達數十公尺，但成本較高、需更換電池。

RFID 與條碼的最大差異在於：RFID **不需要對準讀取器**，可以穿透非金屬包裝讀取，且一次可同時讀取多個標籤，大幅提升作業效率。

**工業應用：物料追蹤與倉儲管理**

RFID 在製造業與物流業的應用極為廣泛：貨架上的零件盒貼有 RFID 標籤，叉車通過貨架時自動讀取所有標籤，完成庫存盤點；零件在生產線上流動時，RFID 記錄其通過每個工站的時間，實現完整的生產追溯；出貨時，貨物通過 RFID 閘門即自動完成出貨記錄，無需人工掃描。

## 九、物聯網與低功耗網路

### 1. IoT 概念

**物聯網（Internet of Things，IoT）** 是指將各種實體物件（機器、家電、車輛、建築設施……）嵌入感測器、處理器與通訊模組，使其能夠蒐集資料並透過網路相互通訊、相互協作的概念。

IoT 的核心思想是「讓物品能夠感知自己的狀態，並與外界溝通」。一台傳統的工廠壓縮機只會運轉；一台 IoT 壓縮機會即時回報自己的溫度、振動、電流、出口壓力，並在數值異常時主動發出警報。

根據連接設備規模的預估，全球 IoT 設備數量正以驚人速度增長，推動著從消費性電子到工業製造的全面數位轉型。

### 2. 低功耗廣域網路（LPWAN）

IoT 設備的特性往往是數量龐大、分布廣泛、電池供電（不能頻繁更換），且每次只傳送少量資料。傳統的 Wi-Fi 或 4G 對這類應用而言功耗太高。為此，專門針對 IoT 需求設計的**低功耗廣域網路（LPWAN）** 技術應運而生：

**LoRaWAN（Long Range WAN）**：能以極低功耗傳輸訊號，覆蓋範圍可達數公里至十幾公里（開闊地形），一顆 AA 電池可支撐感測器運作長達數年。適合農業感測（土壤濕度、氣象）、智慧城市（路燈、停車格）、工廠偏遠角落的感測器。

**NB-IoT（Narrowband IoT）**：基於現有的 4G/5G 蜂巢式網路，由電信業者提供覆蓋，訊號能穿透建築物，適合需要電信業者級別覆蓋的應用（如分布在城市各處的智慧電錶）。

**Zigbee**：低功耗的短距離無線協定，常用於工廠與智慧家庭。Zigbee 裝置可以形成網狀網路，互相中繼訊號，擴大覆蓋範圍，非常適合工廠現場大量感測器的部署。

### 3. 工業應用：設備監測與預測維護

**預測性維護（Predictive Maintenance）** 是 IIoT 最重要的應用之一。透過在設備上安裝振動、溫度、電流等感測器，持續蒐集設備運作資料，再以機器學習模型分析資料模式，預測設備何時可能故障——在故障發生前數天至數週提前維修，避免非計畫停機。

這個流程的實現，完全仰賴可靠的感測器網路：設備上的 IIoT 感測器節點，透過 Zigbee 或 LoRaWAN 將量測資料上傳到工廠閘道器，再透過工廠內網傳入 MES 資料庫，最後被雲端的 AI 分析平台處理，產生維護建議推送給維護工程師的手機。

## 十、低軌衛星與新型網路

### 1. 傳統衛星通訊的限制

傳統的通訊衛星運行在距離地球約 35,786 公里高的**地球靜止軌道（GEO）**。由於距離遙遠，訊號往返一趟（地面 → 衛星 → 地面）的時間約需 **600 毫秒以上**，這個延遲使得 GEO 衛星不適合需要即時互動的應用（如視訊通話、線上遊戲、工業即時控制）。

### 2. 低軌衛星（LEO）概念

**低軌衛星（Low Earth Orbit，LEO）** 運行在距離地球 200–2,000 公里的低軌道。由於距離大幅縮短，訊號往返延遲僅約 **20–40 毫秒**，與地面寬頻網路相近，因此能提供真正可用的寬頻網際網路存取。

由於每顆 LEO 衛星的覆蓋範圍較小（相比 GEO），需要數百至數千顆衛星組成**衛星星座（Constellation）** 才能提供全球持續覆蓋。代表性的 LEO 衛星網路系統包括 SpaceX 的 **Starlink**（目前已部署超過五千顆衛星）與 Amazon 的 Kuiper 計畫。

### 3. 傳統衛星 vs 低軌衛星比較

| 比較項目 | GEO 衛星 | LEO 衛星 |
| --- | --- | --- |
| 軌道高度 | 約 35,786 km | 200–2,000 km |
| 延遲 | 600+ ms | 20–40 ms |
| 覆蓋範圍（單顆） | 全球約 1/3 | 小，需要大量衛星 |
| 衛星數量需求 | 少（3顆可覆蓋全球） | 數百至數千顆 |
| 適用應用 | 廣播、氣象、GPS | 寬頻網際網路、IoT |

### 4. 工業應用：偏遠地區連線與全球供應鏈

LEO 衛星最重要的意義在於：**它為過去無法取得良好網路連線的地區，提供了高速低延遲的網際網路存取**。

對工業工程管理而言，這開啟了幾個新可能：礦場、海上鑽油平台、偏遠農場等無法接通地面光纖的場所，現在可以透過 LEO 衛星接入工業 IoT 平台，實現遠端監控與設備管理。全球供應鏈中，在海上運輸的貨輪可以透過 LEO 衛星即時更新位置與貨況，讓供應鏈可視性延伸到最後一哩。

## 十一、網際網路基本概念

### 1. Internet 的定義

**網際網路（Internet）** 是全球規模的網路系統，由數十億台裝置、數百萬個局部網路（LAN、WAN），透過共同遵守的通訊標準（TCP/IP 協定）互相連接而成。它不是某個組織擁有的單一系統，而是無數個局部網路透過共同語言相互串聯的「網路的網路（Network of Networks）」。

### 2. Network vs Internet 的本質差異

| 面向 | 網路（Network） | 網際網路（Internet） |
| --- | --- | --- |
| 本質 | 局部的連線基礎設施 | 全球互聯的通訊生態系統 |
| 所有權 | 單一組織 | 無單一擁有者 |
| 通訊語言 | 可使用各種協定 | 統一使用 TCP/IP 協定族 |
| 規模 | 有限（一棟建築、一個工廠） | 全球（數十億裝置） |
| 連接方式 | 直接實體連接 | 透過路由器層層轉發 |

一個簡單的比喻：工廠的內部電話系統是「網路」——它讓廠內員工互相通話；而透過公共電話網路打電話到客戶那裡，就需要「網際網路」的概念——跨越不同組織、不同系統的互聯。

### 3. 工業應用：跨企業、跨國資料流

工業企業的資料流動，在今天往往同時涉及「網路」與「網際網路」兩個層次：工廠內部的 PLC 資料透過工廠 LAN 傳入 MES（使用工廠內網）；MES 的生產報表透過企業 WAN 傳到總部（使用廣域網路）；總部的 ERP 同步到雲端 SAP 系統（使用網際網路）；客戶透過網際網路查詢訂單進度（使用公共網際網路）。理解這兩個層次的區分，有助於在設計資訊系統時，正確規劃哪些資料應留在內網、哪些可以上雲端。

## 十二、網際網路簡史

### 1. ARPANET 起源（1960–1970年代）

網際網路的起源可以追溯到1969年，美國國防部高等研究計畫署（ARPA）建立了 **ARPANET**，最初的目的是建立一個即使部分節點被摧毀（如核戰攻擊），資訊仍能透過其他路徑繼續傳遞的韌性通訊網路。

1969年10月29日，ARPANET 完成了人類史上第一次電腦網路通訊：從加州大學洛杉磯分校（UCLA）傳送 "LOGIN" 到史丹佛研究院（SRI），系統在傳到 "O" 時當機，因此第一條網路訊息是 **"LO"**——意外地成了「Hello」的縮寫。

### 2. TCP/IP 發展（1970–1980年代）

早期的 ARPANET 使用多種不相容的通訊協定，不同網路之間難以互聯。1974年，文頓·瑟夫（Vint Cerf）與鮑伯·卡恩（Bob Kahn）共同發表了 **TCP/IP 協定**，提供了一套讓不同網路能夠互相通訊的共同語言。

1983年，ARPANET 全面切換至 TCP/IP，這一天被視為現代網際網路誕生的關鍵節點。TCP/IP 的開放性與通用性，是網際網路能夠無限擴展至今天規模的根本原因。

### 3. WWW 興起（1990年代）

1989年，在歐洲核子研究中心（CERN）任職的英國科學家**提姆·伯納斯-李（Tim Berners-Lee）** 提出了全球資訊網（World Wide Web，WWW）的構想，並於1991年公開。WWW 在 TCP/IP 網路之上，增加了超文字（Hypertext）連結的概念，讓任何人都能透過瀏覽器，以點擊連結的方式在網頁之間漫遊。

1993年，第一個圖形化瀏覽器 **Mosaic** 發布，讓普通人（而非只有工程師）也能輕鬆使用網際網路。1990年代下半，網際網路使用者數量以指數速度增長，電子商務、搜尋引擎、電子郵件成為主流，深刻改變了商業與社會運作。

### 4. 行動網際網路時代（2000年代至今）

2007年，iPhone 的推出開啟了行動網際網路時代。結合 3G/4G/5G 行動通訊、觸控螢幕與 App 生態，網際網路從固定的桌面電腦，延伸到每個人的口袋中，成為隨時隨地存取資訊與服務的基礎設施。

今天，網際網路已從最初的軍事研究工具，演變為支撐全球經濟、文化、教育與社會運作的關鍵基礎設施。

## 十三、封包交換（Packet Switching）

### 1. 封包的概念

在網際網路上，資料並非整塊傳送，而是被切割成許多小的**封包（Packet）**。每個封包都包含：

- **標頭（Header）**：來源 IP 位址、目的地 IP 位址、封包編號、封包大小等控制資訊
- **資料（Payload）**：實際要傳送的內容（檔案的一個片段、網頁的一部分……）

以傳送一份 10 MB 的生產報告為例，這份報告可能被切割成數千個封包，每個封包大小約 1,500 位元組，分別在網路上獨立傳送，最後在目的端重新組合還原成完整的報告。

### 2. 封包傳遞流程

```
來源端                路由器網路                目的端
  │                      │                        │
  ├─ 切割資料為封包 ──────►│                        │
  │                      ├─ 讀取目的地IP ──────────►│
  │                      ├─ 選擇最佳路徑            │
  │                      ├─ 轉發封包 ──────────────►│
  │                      │                        ├─ 接收所有封包
  │                      │                        ├─ 依序號重組
  │                      │                        └─ 還原原始資料
```

每個路由器收到封包後，讀取目的地 IP 位址，根據路由表選擇下一個轉發點，再將封包送出。同一份資料的不同封包，可能走完全不同的路徑到達目的地，最終在接收端按照序號重新組合。

### 3. 封包交換 vs 電路交換

傳統電話網路使用**電路交換（Circuit Switching）**：通話開始時，在發話端與受話端之間建立一條「專屬電路」，通話期間這條電路完全被這通電話佔用，即使沉默無聲時也一樣。通話結束後才釋放電路。

**封包交換（Packet Switching）** 則沒有專屬電路的概念：多個使用者共享同一條網路線路，各自的資料封包依序（或交錯）通過。這帶來了幾個重要優勢：線路使用效率大幅提升（因為沒有資料時，線路可以被其他人使用）；單一鏈路故障時，封包可以自動繞道；也更容易擴展以服務更多使用者。

**工業應用：即時資料傳輸**

工廠的即時監控資料（感測器讀值、設備狀態）透過封包交換網路傳輸時，面臨的挑戰是：由於多個封包共享線路，且各封包可能走不同路徑，**到達時間不一定按照傳送順序**，造成**延遲（Latency）**與**抖動（Jitter）**。對於需要精確時序的工業控制，這是設計時必須考量的因素，通常需要 QoS（服務品質）機制或專用網路來保障關鍵資料的傳輸優先權。

## 十四、通訊協定與分層架構

### 1. 為何需要通訊協定？

**通訊協定（Protocol）** 是通訊雙方必須共同遵守的規則與標準。就像人與人溝通需要說「同一種語言」，電腦與電腦之間也需要協定才能有效溝通。

沒有統一的協定，不同廠商製造的設備將無法互相通訊——就像蘋果手機的充電器不能用於三星手機一樣的困境，只是放大到了全球數十億台裝置的規模。TCP/IP 等開放標準協定，正是讓全球不同廠牌、不同作業系統的裝置能夠無縫互聯的關鍵。

### 2. 分層概念（簡化理解）

網路通訊的複雜功能，被拆解為多個**分層（Layer）**，每一層只負責特定的功能，並向上層提供服務、向下層請求服務。分層設計讓複雜的通訊系統變得可管理——修改某一層的實作，不影響其他層的運作。

以寄一封信的流程來比喻分層概念：

```
【寄信人】 → 寫信（應用層）→ 裝信封（傳輸層）→ 填寫地址（網路層）
              → 投入郵箱（資料鏈結層）→ 郵差取信（實體層）
                                                ↓
【收信人】 ← 讀信（應用層）← 拆信封（傳輸層）← 查地址（網路層）
              ← 從郵箱取出（資料鏈結層）← 郵差送信（實體層）
```

### 3. TCP/IP 架構概覽

現代網際網路使用的 **TCP/IP 協定族**，通常以四層架構描述：

| 層次 | 功能 | 代表協定 | 直觀說明 |
| --- | --- | --- | --- |
| **應用層** | 提供使用者直接使用的服務 | HTTP、HTTPS、FTP、DNS、SMTP | 你實際使用的服務（瀏覽網頁、收發郵件） |
| **傳輸層** | 確保資料可靠或快速送達 | TCP、UDP | 決定「如何寄送」（掛號還是普通信） |
| **網路層** | 負責跨網路的路由與定址 | IP | 決定「走哪條路」送達目的地 |
| **網路存取層** | 在同一網路內的實體傳輸 | Ethernet、Wi-Fi | 實際搬運資料的「貨車與道路」 |

資料從應用程式出發，依序向下通過各層（每層加上該層的控制資訊）；抵達目的端後，依序向上通過各層（每層剝除對應的控制資訊），最終還原原始資料交給應用程式。

## 十五、TCP/IP 基本概念

### 1. IP——定位與位址

**IP（Internet Protocol）** 是網際網路的基礎定址協定，為網路上的每一台裝置分配一個唯一的**IP 位址（IP Address）**，讓資料知道要送往何處。IP 位址就像家庭住址——告訴郵差「你要把這封信送到哪裡」。

IP 協定本身是**不可靠的（Best-Effort）**：它盡力將封包從來源送到目的地，但不保證封包一定到達、不保證順序、不保證沒有重複。可靠性由上層的 TCP 協定來負責。

### 2. TCP——可靠傳輸

**TCP（Transmission Control Protocol，傳輸控制協定）** 在 IP 的基礎上，提供**可靠、有序、無差錯**的資料傳輸。TCP 透過以下機制確保可靠性：

- **三向交握（Three-Way Handshake）**：連線建立前，發送端與接收端先完成三次確認，確保雙方都準備好
- **確認應答（ACK）**：接收端收到每個封包後，回傳確認訊號；若發送端未收到確認，則重傳
- **流量控制（Flow Control）**：根據接收端的處理能力，調整發送速率，避免接收端被資料淹沒
- **擁塞控制（Congestion Control）**：感知網路壅塞時，主動降低發送速率，避免惡化壅塞

TCP 適合需要確保資料完整性的應用：檔案傳輸、網頁瀏覽、電子郵件、資料庫查詢。

### 3. UDP——快速但不保證

**UDP（User Datagram Protocol，使用者資料包協定）** 是一種更輕量的傳輸協定。它不建立連線、不確認收到、不重傳遺失的封包——簡單地將資料包發送出去，不管對方是否收到。

UDP 的優勢是**延遲極低**：省去了 TCP 的各種確認與重傳機制，資料能以最快速度傳送。適合以下應用：即時影音串流（少量畫面遺失可接受，但不能停頓等待重傳）、線上遊戲（低延遲比完整性更重要）、DNS 查詢（簡短的問答，發送一次若沒回應就再發）。

**工業應用：資料可靠性 vs 即時性的取捨**

| 應用場景 | 適合協定 | 原因 |
| --- | --- | --- |
| 生產報表上傳 ERP | TCP | 資料完整性最重要，延遲可接受 |
| 設備即時監控畫面 | UDP | 即時性優先，偶爾一幀遺失可接受 |
| 工業機器人即時控制 | UDP（或專用協定） | 延遲必須極低，重傳會造成危險 |
| 品質量測記錄存入資料庫 | TCP | 每一筆記錄都不能遺失 |

## 十六、IP 位址系統

### 1. IPv4 基本概念

**IPv4（Internet Protocol version 4）** 是目前仍廣泛使用的 IP 位址格式，由 **32 個位元**組成，通常以四組十進位數字表示，每組數字範圍為 0–255，組與組之間以點分隔，例如 `192.168.1.100`。

32 位元的 IPv4 最多能表示 **2³² ≈ 43 億**個不同的位址。在網際網路發展初期，這個數量看似充裕，但隨著連網裝置的爆炸性成長，IPv4 位址已於2011年全球正式耗盡（各地區分配機構陸續宣告用盡）。

**私有 IP 位址（Private IP）** 是只在內部網路使用、不在網際網路上路由的位址範圍，例如 `192.168.0.0/16`（家用路由器常用）、`10.0.0.0/8`（企業內網常用）。透過 **NAT（網路位址轉換）** 技術，多台使用私有 IP 的裝置可以共用一個公共 IP 位址連接網際網路，這是延緩 IPv4 耗盡的主要手段。

### 2. IPv6 發展背景與特性

**IPv6（Internet Protocol version 6）** 是為了解決 IPv4 位址耗盡問題而設計的下一代 IP 協定，使用 **128 個位元**，以十六進位表示，例如 `2001:0db8:85a3:0000:0000:8a2e:0370:7334`。

128 位元的 IPv6 理論上可以提供 **2¹²⁸ ≈ 3.4 × 10³⁸** 個位址——這個數字大到幾乎不可能耗盡，地球上每一粒沙都能分配到數百億個 IP 位址。

除了位址空間大幅擴展，IPv6 還帶來了其他改進：更有效率的路由（簡化的標頭格式）、更好的安全性（IPSec 原生支援）、自動配置（裝置可以自動產生自己的 IPv6 位址）。

### 3. 工業應用：設備數量爆炸下的位址需求

物聯網的普及使連網裝置數量呈指數成長。一個配備完整 IIoT 感測器的現代工廠，可能有數千個需要 IP 位址的終端節點（感測器、PLC、HMI、攝影機、AGV……）。未來的智慧城市與智慧製造，對 IP 位址的需求將更加龐大。IPv6 的超大位址空間，是支撐工業4.0與IoT大規模部署的基礎之一。

## 十七、網址與網域名稱系統（DNS）

### 1. URL 結構

**URL（Uniform Resource Locator，統一資源定位器）** 是在網際網路上精確定位某一資源（網頁、檔案、服務）的標準格式，俗稱「網址」。一個完整的 URL 由幾個部分組成：

```
https://  www.example.com  :443  /products/item  ?id=123  #details
  │              │           │         │             │          │
協定(Protocol) 網域名稱   連接埠   路徑(Path)   查詢參數    錨點
```

- **協定（Protocol）**：說明使用哪種通訊方式存取資源。`http://` 是未加密的超文字傳輸協定；`https://` 是加密版本（Hypertext Transfer Protocol Secure），現代網站應全面採用 HTTPS
- **網域名稱（Domain Name）**：人類可讀的主機識別名稱，如 `www.google.com`
- **路徑（Path）**：伺服器上的資源位置，如 `/products/item`
- **查詢參數（Query Parameters）**：傳遞給伺服器的額外資訊，如 `?id=123`

### 2. DNS 的角色

電腦在網路上只認識 IP 位址，不認識 `www.google.com` 這樣的網域名稱。**DNS（Domain Name System，網域名稱系統）** 扮演的就是「網際網路的電話簿」角色：將人類好記的網域名稱，轉換為電腦能識別的 IP 位址。

DNS 解析的流程：

```
你的瀏覽器輸入 www.google.com
        ↓
查詢本機 DNS 快取 → 若有記錄直接使用
        ↓（若無快取）
詢問 ISP 的 DNS 伺服器
        ↓（若 ISP 也不知道）
逐層向上查詢（根 DNS → .com DNS → google.com DNS）
        ↓
取得 IP 位址（如 142.250.185.46）
        ↓
瀏覽器向該 IP 位址的伺服器發出連線請求
```

整個 DNS 解析過程通常在數十毫秒內完成，對使用者幾乎是無感的。

### 3. 工業應用：雲端平台與設備連接

在工業4.0情境中，工廠設備連接到雲端平台時，也大量使用 DNS。例如，工廠的 IIoT 閘道器要將感測器資料上傳到 AWS IoT Core，它不需要記住 AWS 伺服器的 IP 位址（這個 IP 可能隨時變動），只需要知道 DNS 名稱（如 `iot.us-east-1.amazonaws.com`），透過 DNS 解析找到當前的 IP 位址後建立連線。這種設計讓系統更有彈性——即使雲端平台更換了伺服器 IP，設備端不需要做任何修改。

## 十八、全球資訊網（WWW）與瀏覽器

### 1. WWW 的概念

**全球資訊網（World Wide Web，WWW）** 是建立在網際網路基礎設施之上的一個**超文字資訊系統**。它的核心創新是**超連結（Hyperlink）**：網頁中的文字或圖片可以連結到其他網頁，使用者點擊後立即跳轉，讓資訊之間形成相互連結的「蜘蛛網」。

> **重要區分**：「網際網路（Internet）」是實體的基礎設施（電纜、路由器、協定）；「WWW」是運行在這個基礎設施上的一個服務（網頁與超連結系統）。電子郵件、即時通訊、串流影音也是運行在網際網路上的服務，但不屬於 WWW。

WWW 建立在三項技術標準上：**HTML**（描述網頁內容與結構的語言）、**HTTP/HTTPS**（網頁資料傳輸的協定）、**URL**（定位網頁資源的位址格式）。

### 2. 瀏覽器運作基本流程

當你在瀏覽器輸入一個網址並按下 Enter，背後發生了一系列複雜的步驟：

```
1. DNS 解析：將網域名稱轉換為 IP 位址
2. TCP 連線：與目的伺服器建立 TCP 連線（三向交握）
3. HTTPS 握手：協商加密方式（若使用 HTTPS）
4. 發送 HTTP 請求：告訴伺服器「我要這個網頁」
5. 伺服器處理：伺服器查詢資料庫、執行程式，準備回應
6. 接收 HTTP 回應：伺服器回傳 HTML、CSS、JavaScript 等檔案
7. 瀏覽器渲染：解析 HTML 結構，套用 CSS 樣式，執行 JavaScript
8. 顯示網頁：在螢幕上呈現最終畫面
```

這整個過程通常在 1–3 秒內完成（對使用者可見的部分）。

### 3. 用戶端與伺服器（Client-Server）

WWW 運作在**用戶端—伺服器（Client-Server）** 架構之上：**用戶端（Client）** 是提出請求的一方（你的瀏覽器），**伺服器（Server）** 是回應請求、提供服務的一方（網站的主機）。

這種架構有清楚的職責分工：用戶端負責呈現介面與接受使用者操作，伺服器負責儲存資料與執行業務邏輯。同一個伺服器可以同時服務數千甚至數百萬個用戶端，這正是雲端服務能夠大規模提供服務的基礎。

**工業應用：Web-based 監控系統（Dashboard）**

現代工廠的監控系統越來越多採用 Web-based 架構：MES 或 SCADA 系統提供一個 Web 伺服器，工程師和管理者只需要開啟瀏覽器輸入內部網址，就能看到即時的生產儀表板，無需安裝任何專用軟體。這種設計讓監控系統的維護更簡單（只需更新伺服器端），存取更靈活（任何有瀏覽器的裝置都能使用）。

## 十九、行動裝置與網際網路

### 1. 行動網路與 App 生態

智慧型手機的普及，徹底改變了人類使用網際網路的方式。透過行動數據網路（4G/5G）或 Wi-Fi，手機隨時隨地都能連線，催生了豐富的 **App 生態系**：每個 App 都是一個針對行動裝置優化的軟體，提供從社群媒體、電子商務到工業監控的各種服務。

行動裝置的普及也帶動了 **API（Application Programming Interface，應用程式介面）** 經濟的興起：不同的 App 和服務透過 API 相互呼叫，形成服務之間互相整合的生態系統。例如，工廠的 MES App 可以透過 API 呼叫 ERP 系統取得訂單資料，再透過另一個 API 呼叫天氣服務取得氣象預報，輔助生產排程決策。

### 2. 即時資料與邊緣應用

行動裝置不只是資料的「消費者」，也可以是資料的「蒐集者」。現代智慧型手機配備了 GPS、加速度計、攝影機、麥克風等多種感測器，使其成為強大的行動感測平台。

在工業場合，**邊緣運算（Edge Computing）** 的概念越來越重要：部分資料處理任務不傳回中央伺服器，而是在行動裝置或現場裝置上直接完成，以降低延遲並減少網路頻寬需求。例如，工廠巡檢人員的手機 App 可以利用本機的 AI 模型，對著設備拍照後立即判斷是否有異常，而不需要先上傳照片到雲端再等待回傳結果。

### 3. 工業應用：行動監控與生產管理 App

行動裝置正在改變工廠的管理方式：

- **行動 MES**：生產線上的工人使用工業平板電腦（IP67 防塵防水等級）掃描條碼報工，即時更新生產進度
- **行動維護系統**：設備發生警報時，系統自動推播通知到維護工程師的手機，工程師可在現場用手機查看設備歷史資料、操作手冊，並在完成維修後用手機記錄維修內容
- **管理層儀表板 App**：廠長或高管透過手機 App，隨時查看全廠的 OEE、良率、在製品庫存等 KPI，不受地點限制

## 二十、當代網路議題

### 1. 網路安全（資安基本概念）

隨著網路連線的普及，**資訊安全（Cybersecurity）** 成為所有組織必須認真對待的議題。常見的網路威脅包括：

- **惡意軟體（Malware）**：病毒、木馬、勒索軟體等，入侵裝置後竊取資料或加密勒索
- **釣魚攻擊（Phishing）**：偽裝成可信任的寄件者，誘騙使用者洩漏帳號密碼或點擊惡意連結
- **阻斷服務攻擊（DDoS）**：大量的偽造請求淹沒目標伺服器，使其無法正常服務合法使用者
- **中間人攻擊（Man-in-the-Middle）**：攻擊者暗中攔截通訊雙方的資料，竊聽或竄改

基本的個人資安防護措施：使用強密碼並定期更換、啟用多因素驗證（MFA）、定期更新作業系統與軟體、不隨意點擊可疑連結、使用 HTTPS 加密的網站。

### 2. 工業控制系統（ICS）安全

**工業控制系統（Industrial Control System，ICS）** 的資安（稱為 OT Security）是製造業面臨的獨特挑戰。傳統上，工業控制系統與外部網路實體隔離，被認為天然安全；但工業4.0推動的 OT/IT 整合，讓工業控制系統越來越多連接到企業網路乃至網際網路，使其暴露於更多的資安風險。

工業控制系統一旦遭受攻擊，後果遠比辦公室電腦被駭嚴重：機台控制異常可能造成產品報廢、設備損毀，甚至人員傷亡。2010年的 Stuxnet 事件（針對核電設施 PLC 的蠕蟲攻擊）是工業資安史上的重大警示。

工業資安的基本防護原則包括：網路隔離（IT 網路與 OT 網路分開，中間設防火牆）、最小權限原則（每個帳號只有執行其工作所需的最低權限）、定期稽核（記錄並審查所有系統存取行為）、員工教育訓練。

### 3. 資料隱私與保護

網際網路上的資料傳輸與儲存，涉及大量的個人資料與企業機密。**資料隱私（Data Privacy）** 保護的是個人對自己資料的控制權；**資料保護（Data Protection）** 則是確保資料不被未授權的人存取或使用。

各國已陸續制定相關法規，如歐盟的 **GDPR（一般資料保護規則）**、台灣的**個人資料保護法**，要求企業在蒐集、處理、使用個人資料時，必須取得當事人同意、說明用途、確保安全儲存，並在不再需要時予以刪除。

### 4. 網路壅塞、效能與數位落差

**網路壅塞（Network Congestion）** 發生在網路節點需要處理的資料量超過其容量時，導致封包延遲甚至遺失。在工業環境中，若大量的感測器同時上傳資料（如換班時大量設備同時回報狀態），可能造成網路瞬間壅塞，影響即時監控的即時性。

**數位落差（Digital Divide）** 是指不同地區、不同群體之間在網路存取能力上的差距。在製造業供應鏈的角度，部分供應商因地處偏遠或規模較小而缺乏可靠的高速網路，可能成為供應鏈數位化的瓶頸。LEO 衛星網路（如 Starlink）的出現，為縮小地理數位落差提供了新的可能。

## 二十一、從設備到雲端的資料流

### 1. 完整的資料旅程

在工業4.0情境中，一筆感測器資料從產生到最終支援管理決策，要歷經多個層次的網路傳輸。以工廠內一台 CNC 加工機床的主軸溫度感測器為例：

```
【感測器】
  溫度值 23.5°C（類比訊號）
        ↓ ADC 轉換
【PLC / 邊緣裝置】
  數位值 23.5，透過 Modbus 傳至 PLC
        ↓ 工業乙太網路（工廠 LAN）
【工廠 MES 伺服器】
  存入資料庫，計算 OEE，觸發異常判斷
        ↓ 企業 WAN / VPN
【企業總部 ERP 系統】
  整合生產成本，更新產能計畫
        ↓ 網際網路（HTTPS）
【雲端分析平台】
  跨廠區大數據分析，AI 模型預測設備壽命
        ↓ 推播通知
【工程師的手機 App】
  收到預警：「M-003 主軸溫度異常，建議本週進行預防性維護」
```

每一個「↓」箭頭都代表一段網路傳輸，使用了不同的網路類型（工廠 LAN、企業 WAN、公共網際網路）與不同的協定（Modbus、TCP/IP、HTTPS）。

### 2. 資料整合的挑戰

在實際工廠中，這條資料旅程面臨幾個常見的挑戰：**協定異構**（不同廠商的設備說不同的工業語言，需要閘道器轉換）；**網路可靠性**（工廠環境的電磁干擾可能造成訊號不穩）；**資安邊界**（工廠內網與企業外網的交界需要嚴格管控）；**資料時序**（來自不同設備的資料需要統一時間戳記，才能做有意義的關聯分析）。

## 二十二、智慧製造網路架構（概念性）

### 1. 四層架構

現代智慧工廠的網路與資訊架構，通常以「四層」模型來描述，從底層的物理設備到頂層的雲端分析，每一層有其專屬的功能與通訊需求：

**第一層：現場層（設備層）**

這是最接近物理世界的一層，包含生產機台、感測器、致動器、機器人等實體設備。這一層的通訊特點是：**即時性要求極高**（毫秒甚至微秒級）、資料量大但每筆資料小、使用工業專用通訊協定（Modbus、PROFIBUS、EtherNet/IP）。

**第二層：控制層（PLC / SCADA 層）**

PLC 從現場層設備蒐集感測資料，執行即時控制邏輯，並將資料彙整後向上傳送。SCADA 系統在這一層提供操作員的監控介面。通訊需求仍需要低延遲，但可容許稍多的資料量，使用工業乙太網路（PROFINET、EtherNet/IP）。

**第三層：管理層（MES / ERP 層）**

MES（製造執行系統）從控制層接收生產資料，計算 KPI（OEE、良率、稼動率），管理工單排程與品質追溯，並與 ERP 系統交換計畫與成本資料。這一層使用標準的 TCP/IP 企業網路，延遲要求相對寬鬆（秒級）。

**第四層：雲端層（分析與決策層）**

雲端平台接收來自管理層的生產數據，執行跨廠區的大數據分析、AI 模型訓練與推論（預測性維護、需求預測）、企業級儀表板呈現。這一層透過網際網路連接，使用 HTTPS、MQTT 等標準網路協定。

```
第四層  ┌─────────────────────────────────────────┐
雲端層  │   雲端平台（AWS / Azure / 工業IoT平台）   │
        │   大數據分析  ◆  AI 模型  ◆  BI 儀表板  │
        └───────────────────┬─────────────────────┘
                  網際網路（HTTPS/MQTT）
第三層  ┌───────────────────┴─────────────────────┐
管理層  │         MES 伺服器  ◆  ERP 系統          │
        │       生產排程  ◆  品質管理  ◆  成本追蹤  │
        └───────────────────┬─────────────────────┘
               企業內網（TCP/IP Ethernet）
第二層  ┌───────────────────┴─────────────────────┐
控制層  │        PLC 控制器  ◆  SCADA 系統         │
        │     即時控制邏輯  ◆  操作員監控介面      │
        └───────────────────┬─────────────────────┘
            工業乙太網路（PROFINET / EtherNet/IP）
第一層  ┌───────────────────┴─────────────────────┐
現場層  │  CNC機床  感測器  機器人  輸送帶  AGV   │
        │         各種工業現場設備                  │
        └─────────────────────────────────────────┘
              工業現場匯流排（Modbus / PROFIBUS）
```

## 二十三、工業4.0中的網路角色

### 1. 即時連線（Real-time Connectivity）

工業4.0的核心承諾之一，是讓製造系統中的每個元素——機台、零件、人員、訂單——都能夠即時感知彼此的狀態，並根據最新資訊做出協調。這種**即時連線能力**是所有其他工業4.0技術的基礎。

沒有可靠的即時網路連線，預測性維護的感測資料無法即時回傳、AGV 無法與生產排程系統即時協調、數位孿生無法即時反映實體設備的狀態。**網路品質（頻寬、延遲、可靠性）直接決定了工業4.0系統能夠達到的能力上限**。

### 2. 資料驅動決策（Data-Driven Decision Making）

在傳統工廠，決策往往依賴管理者的經驗判斷，輔以定期（日報、週報）的統計數字。工業4.0透過網路實現的資料即時蒐集與傳輸，使得**決策依據從「歷史統計」轉向「即時資料」**。

一個典型的轉變：以前工廠每個月統計一次設備的稼動率，下個月才能知道上個月的問題；現在透過即時監控網路，管理者在問題發生的當下就收到警報，能在幾分鐘內啟動應對措施，而不是等到月底報告才發現問題。

### 3. 系統整合（Integration）

工業4.0的另一個核心是**水平整合**（同一工廠內各系統之間的整合）與**垂直整合**（從現場設備到雲端管理平台的整合）。網路是實現這些整合的媒介。

**水平整合範例**：倉儲系統察覺某物料庫存低於安全庫存，透過網路自動觸發 ERP 的採購申請，同時通知生產排程系統調整生產優先序。這一系列的自動化協調，全靠系統間的網路連線與資料交換。

**垂直整合範例**：現場感測器的溫度異常，經由 PLC 判斷後立即調整製程參數（毫秒級），同時將事件記錄透過 MES 存入資料庫（秒級），最終在雲端平台觸發預測模型重新評估（分鐘級）。三個層次的反應，分別在不同的時間尺度下發生，共同確保製程品質。

> **小結**：網路不只是工業4.0的「配件」，而是使能工業4.0的核心基礎設施。理解不同網路技術的特性（速度、延遲、可靠性、覆蓋範圍、成本），是工業工程管理者在規劃與評估智慧製造系統時不可或缺的知識。

## 二十四、學習重點總結

完成本課程後，你應該能夠理解以下核心概念，並將其應用於工業場域的思考與決策：

**網路（Network）是連線技術與架構**

從網路拓樸的選擇（星狀、匯流排、環狀、網狀）、硬體設備的功能（NIC、交換器、路由器、閘道器），到有線（乙太網路）與無線（Wi-Fi、藍牙、Zigbee、LoRaWAN）的技術特性，再到 5G 與 LEO 衛星等新興連線技術——這些構成了支撐現代工廠智慧化的實體基礎。

**網際網路（Internet）是跨網路的全球通訊系統**

網際網路的本質是「網路的網路」，靠 TCP/IP 協定族讓全球不同系統能夠溝通。理解封包交換的概念、TCP（可靠）與 UDP（快速）的取捨、IP 位址的定址機制、DNS 的名稱解析，以及 WWW 的超文字架構，有助於理解工業系統如何安全地連接到雲端服務。

**封包、協定與 IP 位址的基本概念**

資料在網路上以「封包」的形式傳輸，每個封包都攜帶目的地 IP 位址，由路由器逐跳轉發。協定是通訊的共同語言，分層設計讓複雜的網路功能變得可管理。這些概念是理解任何網路系統行為的基礎。

**現代網路如何支撐工業4.0與智慧製造**

從感測器的 LoRaWAN 低功耗連線、到 PLC 的工業乙太網路、到 MES 的企業內網、到雲端平台的網際網路連線——工業4.0的「垂直整合」依賴不同層次的網路技術協同運作。5G 私有網路的超低延遲，正在使以往不可能的即時工業控制成為現實。

**網路技術如何影響生產、物流與管理決策**

RFID 讓倉儲盤點從數天縮短為數小時；行動監控 App 讓管理者隨時掌握全球工廠狀況；LEO 衛星讓偏遠地區的工廠也能接入工業 IoT 平台；OT 資安的防護確保生產控制系統不受外部攻擊威脅。網路技術的選擇，直接影響企業的競爭力與風險管理能力。
